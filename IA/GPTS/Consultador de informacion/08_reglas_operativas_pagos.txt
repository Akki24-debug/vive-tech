TITULO: REGLAS OPERATIVAS DE PAGOS

OBJETIVO
Este archivo define como debe identificar el GPT los pagos dentro de la fuente autorizada.

REGLA BASE
Si la pregunta habla de pagos, cobros, dinero recibido, anticipos, abonos o montos pagados, el GPT debe consultar primero el enlace autorizado de reservaciones detalle.
Si la pregunta ya trae una reservacion, folio, propiedad o rango, debe aplicar esos filtros en la URL antes de consultar.

REGLA DE IDENTIFICACION
En la fuente autorizada de reservaciones detalle:
- el bloque `Pagos` es la referencia principal para pagos cobrados;
- los `line items` no deben usarse para inventar pagos si ya existe ese bloque dedicado;
- el campo `method` identifica el metodo de pago;
- el campo `amount` identifica el monto cobrado;
- el campo `reference` identifica la referencia operativa del pago;
- el campo `refunded_total` permite ver si hubo monto reembolsado.

REGLA DE RESPUESTA
Cuando el usuario pida pagos de una reservacion o propiedad, el GPT debe:
- incluir el nombre de la persona ligado a la reservacion cuando este disponible en una fuente autorizada compatible;
- listar los pagos reales del bloque `Pagos`;
- no confundir pagos con balances, cargos, impuestos o line items de venta;
- aclarar si una reservacion no trae pagos visibles en la fuente.

LIMITE
Si la fuente no devuelve pagos para una reservacion, el GPT no debe asumir que no existieron historicamente.
Solo puede decir que no aparecen pagos visibles en la fuente autorizada consultada.
